Roadmap

Dashboards

Every folder can include sets of dashboards(pages). Dashboard is built by drag & drop tables or table views. It can also contain static data(titles, etc.) and simple expressions.
Dashboards are updated in real-time.

Remote databases

Right now, all SkyAlt data are inside the build-in database. This feature will add the option to connect to remote databases(MySQL, PostgreSQL, etc.) through ODBC.
A user defines columns format(menu, checkbox, etc.) and be able to set/change data(not structure) in the original database. Users can also create views.
Data from remote databases are cached in SkyAlt and re-download if needed.

Time machine

Users can go back in time and restore the project. Something like a Git. SkyAlt file system is already built that way(when a user hits "save", it saves only changes), so the only thing missing is UI.

Views

Map

Rich-text

Every text can be formated + multi-lines.
For formating SkyAlt will use Markdown markup language.

Updates

SkyAlt checks If there is new version everytime is started and then every x hours/minutes. Can be disabled in settings.

Templates

Library with project examples to show a high range of use-cases. Users can use them as a start point or learn from them.

API

Run HTTPS server in SkyAlt so other programs can connect and extract/write data.

Automations

You shouldn't repeat the same steps in SkyAlt over and over. Why not record those steps and then trigger them by pressing button, action or timer? Steps can be applied on one record or billions across different tables.

Data entry

Limits manual data entry to a minimum. Copy/paste and drag & drop are ok because it moves data precisely where you want, but writing text to cell is a problem.
The solution is probably scrapers(web, email, etc.).

Files preview

Video
Audio
Pdfs
Docs
As many others formats as possible

File Search

Right now, SkyAlt searches only text/numbers in cells, but not in files in cells.
Support for HTML, DOC, XLS, PDF, text in images, etc..

Cloud

Privacy is an important part of this project so cloud computing is great challenge. The best solution is to do computing on-premise. Data can be on-premise or end-to-end encrypted on a server(decryption key stays on clients devices) where a client can access them 24/7.
On-premise server version can be in a container(for eg. Docker and Kubernetes).
